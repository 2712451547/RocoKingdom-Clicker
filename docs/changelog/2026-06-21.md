# 更新日志 — 2026-06-21

记录 2026-06-21 的变更，供发布说明与回溯使用。

主要变更
- **Interception DLL 自动定位**：`InterceptionCore.py` 启动时按架构（64 位 Python 优先 `x64`，否则 `x86`）自动在 `third/Interception/library/{arch}/interception.dll`、项目根目录、当前工作目录下查找 DLL，不再需要手动放到特定位置。
- **驱动/DLL 未就绪时弹窗提示**：`InterceptionCore.py` 新增 `init_error` 字段和 `is_ready()` 方法；`Clicker.py` 启动时会调用 Windows `MessageBoxW` 弹出置顶提示，明确告知用户需要以管理员身份运行 `driver_installer\install-interception.exe /install` 安装驱动并重启系统。GUI 中点击"启动"或执行脚本若驱动未就绪，同样会弹出提示而非静默失败。
- **run_clicker.vbs 编码修复**：将 VBS 启动脚本中的中文字符改为英文，避免 `Windows Script Host` 因 UTF-8 文件编码与系统 GBK 编码不匹配报"未结束的字符串常量"（错误码 `800A0409`）。脚本新增 DLL / 驱动安装程序预检逻辑，缺失时给出明确提示。
- **run_clicker.bat 启动器**：命令行启动脚本同样增加 DLL / 驱动预检，优先启动发布目录下的 `RocoKingdom_Clicker.exe`，缺失时回退到 `.venv\Scripts\pythonw.exe` 或系统 `pythonw.exe`，并使用 PowerShell `Start-Process -Verb RunAs` 申请管理员权限。
- **开源许可证配置**：按 `third/Interception/licenses/` 中的第三方许可（LGPL-3.0 + 商业授权备选），在项目根目录新增三份许可证文件：
  - `LICENSE` — 本项目原创 Python/脚本/文档代码采用 **MIT License**。
  - `COPYING` — **GNU General Public License v3.0**（LGPL-3.0 要求随附）。
  - `COPYING.LESSER` — **GNU Lesser General Public License v3.0**（Interception 的实际许可）。
  - `README.md` 末尾新增"许可协议 (Licensing)"章节，说明三方许可关系与分发注意事项。
- **`build_release.bat` 构建流程改进**：把文件复制步骤（数据目录、DLL、驱动安装程序、三份许可证、third-party 许可证目录、README、启动脚本）统一抽离到 `_release_copy.ps1`，避免 cmd/PowerShell 混合执行导致的复制静默失败。构建产物 `release/RocoKingdom_Clicker.zip` 现在完整包含上述所有文件。
- **图片资源优化**：`docs/pic/support.png`（QQ 交流群二维码）从 1284×2280 缩小至 642×1140（各边 50%），保持可读的同时显著减少 Markdown 渲染时的占位空间。
- **输入录制功能**：新增 `InputRecorder.py`（捕获线程 + 热键监听 + 调试浮窗）、`PlaybackEngine.py`（回放引擎）；录制时事件实时显示在浮窗中，且事件会被发回系统（`interception_send`）而非截断，F8/F9 停止热键始终可用；录制脚本保存在 `data/action_scripts/recorded_{name}_{timestamp}.json`。
- **Interception API 修正**：`InterceptionCore.py` 补全所有 `interception_*` 函数的 `argtypes`/`restype` 类型声明；`interception_set_filter` 的谓词参数改为 `CFUNCTYPE` 包装后的函数指针；鼠标按钮位在 `state` 而非 `flags`；键盘 `state` 0=down/1=up 而非 `0x80`。
- **GUI 脚本列表统一 + 进度条**：`webview.py` 合并动作脚本与录制脚本为单一列表，以 `⚙`（动作）/ `🎙`（录制）图标区分；双击任一脚本均可执行（按 `meta.type` 分流至对应引擎）；新增进度条显示录制脚本回放进度（`事件 N / M`）；录制列表高度扩大至撑满可用区域。
- **脚本格式统一 + 互播放**：`ActionScript.save_script()` 保存时写入 `meta.type="action"`；`ActionScript.list_scripts()` / `gui.list_recorded_scripts()` 分别只返回对应类型的脚本；`gui.play_recording()` 按 `meta.type` 将录制脚本路由至 `PlaybackEngine`、动作脚本路由至 `ActionScript._run_script_session`，实现双向互播。

## 二次更新（录制功能修复 + UI 重构）

### 录制功能核心修复

- **脚本列表映射方案**：`webview.py` 新增 `self._script_map: dict[str, tuple[str, str]]`，维护「显示名 → (stem, 类型)」映射，替代之前不可靠的字符串分割方案。`_selected_script()` 直接查映射返回 `(stem, type)`，彻底解决执行按钮无反应的问题。
- **F8/F9 热键扫描码修正**：`InputRecorder.py` 的 `SCANCODE_NAMES` 表修正为正确的 AT Set 2 扫描码（F7=0x41, F8=0x42, F9=0x43），之前 F7/F8/F9 全部错位导致热键被吞进录制内容。录制循环内检测到热键扫描码时触发 `on_stop`/`on_cancel` 回调。
- **录制闪退修复**：`DebugOverlay` 不再在子线程创建 `tk.Tk()` 实例（会崩溃），改为通过 `on_event` 回调把事件计数推送给主 GUI 的 toast。
- **异步发送线程**：`InputRecorder.py` 新增独立 `_send_loop` 线程处理 `interception_send`，避免 send 阻塞导致录制循环来不及 receive 而漏事件。
- **批量 receive**：录制循环改用 `interception_receive(ctx, device, batch_buf, BATCH_SIZE)` 一次取 64 个事件，避免驱动队列积压。
- **鼠标移动事件判断修正**：`MOUSE_MOVE` 不是 state 位（是 filter 位 0x1000），鼠标移动事件 `state=0`，通过 `flags` 区分相对/绝对移动。录制时只记录相对移动（`flags=0` 且 x/y 非零）。
- **记录初始鼠标坐标**：`InputRecorder.start()` 用 `GetCursorPos` 记录屏幕坐标，保存到 meta 的 `start_cursor_x/y`；回放时先用 `SetCursorPos` 恢复初始位置，保证完全复现。
- **回放 move 事件修正**：`PlaybackEngine._build_mouse_stroke()` 中 move 事件设为 `state=0, flags=0`（相对移动），之前错误地设为 `state=0x800`（那是 HWHEEL 水平滚轮！）。
- **不禁用 Windows 鼠标加速**：移除回放时禁用鼠标加速的逻辑。Interception 捕获的是驱动层原始值（加速前），`interception_send` 注入后也会经过 Windows 鼠标处理管线。录制和回放必须用相同的 Windows 设置，才能让相同的 dx/dy 产生相同的光标位移。
- **F2 暂停回放**：`PlaybackEngine` 新增 `pause()`/`resume()`/`toggle_pause()`/`is_paused()`，回放中按 F2 切换暂停/恢复。
- **所有异常路径弹 toast**：录制/回放的所有失败路径都通过 toast 提示用户，不再静默失败。

### 回放完成回调

- `PlaybackEngine` 新增 `on_complete: Callable[[bool, bool], None]` 回调（签名 `callback(success, stopped)`），在 `_playback_loop` 的 `finally` 块中调用。
- `ClickerManager` 设置回调为 `_on_playback_complete`，回放结束时弹 toast：正常完成 `✅ 回放完成` / 被停止 `⏹ 回放已停止` / 异常 `❌ 回放异常`。

- **状态文字动态变色**：运行中(绿)/暂停(橙)/已停止(灰)，根据状态实时切换颜色。
