# 更新日志 — 2026-06-21

记录 2026-06-21 的变更，供发布说明与回溯使用。

主要变更
- **Interception DLL 自动定位**：`InterceptionCore.py` 启动时按架构（64 位 Python 优先 `x64`，否则 `x86`）自动在 `third/Interception/library/{arch}/interception.dll`、项目根目录、当前工作目录下查找 DLL，不再需要手动放到特定位置。
- **驱动/DLL 未就绪时弹窗提示**：`InterceptionCore.py` 新增 `init_error` 字段和 `is_ready()` 方法；`Clicker.py` 启动时会调用 Windows `MessageBoxW` 弹出置顶提示，明确告知用户需要以管理员身份运行 `driver_installer\install-interception.exe /install` 安装驱动并重启系统。GUI 中点击"启动"或执行脚本若驱动未就绪，同样会弹出提示而非静默失败。
- **run_clicker.vbs 编码修复**：将 VBS 启动脚本中的中文字符改为英文，避免 `Windows Script Host` 因 UTF-8 文件编码与系统 GBK 编码不匹配报"未结束的字符串常量"（错误码 `800A0409`）。脚本新增 DLL / 驱动安装程序预检逻辑，缺失时给出明确提示。
- **run_clicker.bat 启动器**：命令行启动脚本同样增加 DLL / 驱动预检，优先启动发布目录下的 `RocoKingdom_Clicker.exe`，缺失时回退到 `.venv\Scripts\pythonw.exe` 或系统 `pythonw.exe`，并使用 PowerShell `Start-Process -Verb RunAs` 申请管理员权限。
- **开源许可证配置**：按 `third/Interception/licenses/` 中的第三方许可（LGPL-3.0 + 商业授权备选），在项目根目录新增三份许可证文件：
  - `LICENSE` — 本项目原创 Python/脚本/文档代码采用 **MIT License**。
  - `COPYING` — **GNU General Public License v3.0**（LGPL-3.0 要求随附）。
  - `COPYING.LESSER` — **GNU Lesser General Public License v3.0**（Interception 的实际许可）。
  - `README.md` 末尾新增"许可协议 (Licensing)"章节，说明三方许可关系与分发注意事项。
- **`build_release.bat` 构建流程改进**：把文件复制步骤（数据目录、DLL、驱动安装程序、三份许可证、third-party 许可证目录、README、启动脚本）统一抽离到 `_release_copy.ps1`，避免 cmd/PowerShell 混合执行导致的复制静默失败。构建产物 `release/RocoKingdom_Clicker.zip` 现在完整包含上述所有文件。
- **图片资源优化**：`docs/pic/support.png`（QQ 交流群二维码）从 1284×2280 缩小至 642×1140（各边 50%），保持可读的同时显著减少 Markdown 渲染时的占位空间。

变更影响的文件（摘要）
- `InterceptionCore.py`：新增 `is_ready()`、`init_error` 字段，`_initialize_interception()` 不再抛异常，所有错误路径写入 `init_error` 供上层展示。
- `Clicker.py`：`main()` 启动前做驱动预检并通过 `user32.MessageBoxW` 弹窗；`ClickerManager` 新增 `_show_driver_warning()` 与"启动/执行脚本"入口的驱动检查。
- `gui.py`：状态栏 `get_status()` 新增 `interception_ready`、`interception_error` 字段；GUI 启动时若驱动未就绪会弹出温和提示。
- `run_clicker.vbs`：整体改英文提示，新增 DLL / 驱动安装程序预检逻辑。
- `run_clicker.bat`：新增启动预检与多种启动路径回退。
- `build_release.bat`：简化为 Python 环境探测 + PyInstaller 调用 + 调用 `_release_copy.ps1` 完成文件复制与打包。
- `_release_copy.ps1`（新）：集中处理发布目录的文件复制、第三方许可证携带、zip 归档。
- `LICENSE`、`COPYING`、`COPYING.LESSER`（新）：三份许可证文本。
- `docs/pic/support.png`：分辨率缩小到原来的 50%。

快速验证步骤
1. 双击 `dist\RocoKingdom_Clicker\run_clicker.vbs`：不再报"未结束的字符串常量"；若 DLL 或驱动安装程序缺失会弹出英文提示。
2. `python Clicker.py --gui` 手动启动：若 Interception 驱动未安装或 `interception.dll` 找不到，会先弹出置顶的 MessageBox 提示并退出，而非静默崩溃。
3. 运行 `build_release.bat` 后检查 `dist\RocoKingdom_Clicker`：应包含 `LICENSE`、`COPYING`、`COPYING.LESSER`、`interception.dll`、`driver_installer\install-interception.exe`、`third_party_licenses\Interception\`、`run_clicker.vbs`、`run_clicker.bat`、`README.md`、`data\`。
4. 查看 README.md 中"![QQ交流群](docs/pic/support.png)" — 图片占用的高度约为原来的一半。

调试与注意事项
- `InterceptionCore.is_ready()` 仅检查 DLL 是否可加载且 `interception_create_context()` 是否成功；驱动服务（`interception` 内核驱动）需通过 `driver_installer\install-interception.exe /install` 安装并**重启**后才会生效。
- 运行 `Clicker.py` 或 `RocoKingdom_Clicker.exe` 时必须为管理员权限，否则即使驱动已安装也可能无法注入键盘事件。
- `run_clicker.vbs` / `run_clicker.bat` 会通过 `ShellExecute("runas")` 自动申请 UAC 提权。
- `_release_copy.ps1` 需配合 `build_release.bat` 使用，单独执行时需要 `dist\RocoKingdom_Clicker\RocoKingdom_Clicker.exe` 已存在。
