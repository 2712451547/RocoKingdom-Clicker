# 更新日志 — 2026-06-29

记录 2026-06-29 的变更，主要内容：暂停/继续/停止控制体系、可自定义热键系统、UI 布局重构。

## 主要变更

### 1. 暂停/继续/停止控制体系

- **新增暂停/继续/停止按钮**：所有脚本（录制回放 + 自定义动作脚本）统一支持暂停、继续、停止三个操作。按钮位于执行按钮旁，分色区分：
  - `⏸ 暂停`（橙色 `Warning.TButton`）
  - `▶ 继续`（绿色 `Success.TButton`）
  - `⏹ 停止`（红色 `Danger.TButton`，直接终止当前脚本）
- **按钮状态动态管理**：根据 `script_running`/`script_paused`/`playback_active`/`playback_paused` 自动启用/禁用。运行中可暂停、可停止；暂停中可继续、可停止；空闲时全部禁用。
- **暂停时切换脚本确认弹窗**：如果当前脚本已暂停，用户点击列表切换到其他脚本时，弹出 `askyesno` 确认框，告知"切换将停止当前脚本且不保留执行进度"。用户取消则恢复选中项；用户确认则停止当前脚本。
- **执行时冲突确认**：有脚本/回放正在运行或暂停时，执行其他脚本会先弹确认框，确认后停止当前任务再执行新脚本。
- **列表刷新抑制选择事件**：刷新脚本列表期间设置 `_suppress_select_event` 标志，避免恢复选中项时误触发切换确认弹窗。

### 2. 可自定义热键系统（F1-F12）

- **热键从硬编码改为可配置**：暂停/继续、开始录制、停止录制并保存、取消录制四个热键全部支持用户自定义，范围 F1-F12，防止系统占用用户想录制的按键。
- **热键配置持久化**：新增 `data/clicker_configs/hotkeys.json`，`ConfigManager.load_hotkeys()`/`save_hotkeys()` 负责读写。默认配置：F2 暂停/继续、F7 开始录制、F8 停止保存、F9 取消。
- **全局热键分发线程**：`Clicker.py` 新增 `_hotkey_dispatch_loop` 后台线程，持续监听 `GlobalHotkeyListener` 事件队列，根据当前热键配置分发到对应处理函数。替代原来分散在 `_run_script_session` 和 `listen_hotkeys` 中的热键处理逻辑。
- **热键冲突检测**：
  - `ClickerManager.set_hotkeys()` 对全部 4 个热键做全局判重，冲突时 toast 提示具体冲突按键（如"❌ 按键冲突：F7 被多个功能使用"）。
  - `InputRecorder.set_hotkeys()` 新增 start/stop/cancel 三键互斥检查，返回 `bool`，冲突时拒绝修改并记录警告日志。
- **InputRecorder 热键可配置化**：`HOTKEY_START`/`HOTKEY_STOP`/`HOTKEY_CANCEL` 从类常量改为实例属性，通过 `set_hotkeys()` 方法运行时修改。

### 3. UI 布局重构

- **左栏拆分为三组独立 LabelFrame**：
  - **状态信息**（只读）：当前选择脚本 + 状态显示
  - **全局控制选项**：控制自定义脚本鼠标移动复选框
  - **输入录制**：录制名输入框 + 录制/保存/取消按钮
- **删除运行状态栏**：原 4 栏布局精简为 3 栏（左栏 / 脚本列表 / 快捷键）。
- **热键设置面板**：右栏新增快捷键设置区，每个热键一行（功能名 + F1-F12 下拉选择框）+ 保存按钮。
- **当前设置显示区改造**：从丑陋的 `tk.Text` 文本框改为彩色按键徽章（左侧灰色功能名 + 右侧蓝色圆角按键标签，白字粗体），与设置区风格统一。
- **底部 footer 动态显示**：单行横向排列当前热键配置，如 `快捷键：F2 暂停/继续 | F7 开始录制 | F8 停止录制并保存 | F9 取消录制`。
- **按钮 label 标注热键**：按钮文字动态显示对应热键，如 `⏸ 暂停 (F2)`、`● 录制 (F7)`、`■ 保存 (F8)`、`✕ 取消 (F9)`。热键修改保存后自动更新。
- **取消按钮样式统一**：新增 `Secondary.TButton` 样式（灰底白字粗体），取消按钮从默认 `TButton` 改为该样式，与录制/保存按钮字体统一。

### 4. 快捷键 GUI 交互修复

- **下拉框被刷新循环覆盖问题**：引入脏标记机制。用户修改下拉框时（`<<ComboboxSelected>>` 事件）设置 `_hotkey_dirty = True`，`refresh_status` 同步下拉框时若脏标记为 True 则全部跳过；保存成功后清除脏标记并立即刷新。彻底解决用户选好值还没点保存就被后端旧值覆盖的问题。
- **焦点检查兜底**：下拉框有焦点时也跳过同步，作为脏标记的补充保护。

### 5. 录制脚本名格式优化

- **文件名格式调整**：从 `recorded_{name}_{timestamp}.json` 改为 `{name}_{日期}_{时间}.json`。
  - 去掉 `recorded_` 前缀，直接用用户输入的脚本名作为前缀。
  - 日期和时间分开显示，格式 `yyyymmdd_HHMMSS`。
  - 改用本地时间生成文件名（用户更直观），UTC 时间仍记录在 `meta.recorded_at`。
  - 示例：用户输入"我的测试" → `我的测试_20260623_235229.json`。

## 快速验证步骤

1. 启动 GUI（`python gui.py`），确认 3 栏布局正常：左栏三组（状态信息 / 全局控制选项 / 输入录制）、中栏脚本列表、右栏快捷键面板。
2. 在快捷键面板修改某个热键（如暂停/继续改为 F3），点击"保存热键设置"，确认：
   - 弹出"✓ 热键配置已保存"toast
   - 按钮 label 自动更新为 `⏸ 暂停 (F3)` / `▶ 继续 (F3)`
   - 当前设置徽章更新为 F3
   - 底部 footer 更新
   - 重启程序后配置仍生效（持久化）
3. 尝试把两个功能设为同一按键（如暂停和录制都设 F5），保存时应弹出冲突提示。
4. 执行一个脚本，确认暂停/继续/停止按钮状态正确响应：运行中可暂停可停止，暂停中可继续可停止。
5. 脚本暂停时点击列表其他脚本，确认弹出切换确认弹窗。
6. 录制一段输入并保存，确认文件名格式为 `{输入名}_{日期}_{时间}.json`。

## 调试与注意事项

- 热键配置文件位于 `data/clicker_configs/hotkeys.json`，删除该文件可恢复默认配置。
- 全局热键分发线程为 daemon 线程，程序退出时自动结束。
- `InputRecorder.set_hotkeys()` 的互斥判定是独立于 `ClickerManager.set_hotkeys()` 的二次防护，即使直接调用 recorder 也不会接受冲突配置。
- 暂停/继续/停止 API 在子线程执行（通过 `gui.py` 的 worker 线程），避免阻塞 UI。
- 脏标记机制确保用户从修改下拉框到点击保存期间，下拉框值不会被 500ms 刷新循环覆盖。
